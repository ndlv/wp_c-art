
Bonjour à toute l'équipe, j'éssaye de trouver un moyen pour que ceux qu'il le veulent puisse
utiliser wordpress sur le serveur local sans avoir de conflicts avec les versions des fichiers des 
autres membres du groupe.

Grace à cette page je tente une nouvelle méthode de travail : 
voici le lien : http://plausiblethought.net/wordpress-git-workflow/

Alors je vous donne quelques explications maintenant : 

Pour Wordpress fonctionne il a besoin d'un certain nombre de fichiers qui le configure.
Tout ces fichier qui représente  le coeur de l'application ne seront pas sur le repository git
mais dans chacun de nos dossier de travail respectif pour que l'on puisse faire tout fonctionner.
Nous ne partageons que les fichier qui font évoluer le site, pas les fichiers qui font tourner wordpress.

Pour ce faire j'utilise un petit fichier qui s'appelle .gitignore . Par défaut ce fichier est caché dans le finder. Affichez les fichiers et dossier cachés si vous voulez le voir. Dans ce fichier on indique tout les type de fichier que l'on ne veut pas envoyer par le biais de git.

Par exemple si je met dans le git ignore  *.o  cela veut dire que tout les fichier avec l'extension
o ne seront pas envoyé. Voila l'idée. Vous n'avez pas à toucher à ce gitignore normalement.

CONSEQUENCE ------->

Lorsque vous allez cloner le repository et récuperer les fichier de wordpress, votre dossier de travail sera incomplet et vous ne pourrez pas faire tourner wordpress sur votre serveur local.

Vous devez donc ajouter les fichiers manuellement, je donne la méthode qui est expliqué dans la page 
que j'ai mi en lien au dessus.

1. Ouvrez le terminal
2. Rendez vous dans le dossier de wordpress

cd  chemin_vers_votre_dossier_de_travail/.../wp_blocarte/
